feat(ansible): Complete core migration to Ansible

This commit fully migrates the core automation logic of the project to Ansible.

- **All Docker-based Stacks:** Generic, reusable Ansible roles created for `proxy`, `media`, `files`, `webtools`, and `monitoring` stacks.
- **Special Stacks:** Dedicated Ansible roles created for `backup` (PBS installation) and `development` (control node setup) stacks.
- **Host Configuration:** The `proxmox_host_setup` role replaces `helper-menu.sh`'s functionality (timezone, security, storage, ZFS optimization, network bonding).
- **Automated Installation:** The `installer.sh` script is updated to fully automate the control node setup and Proxmox API token creation, securely injecting secrets.
- **Menu System:** The `main-menu.menu.sh` script is transformed into an Ansible-centric menu that calls the new playbooks.
- **Old Script Cleanup:** `fail2ban-manager.sh` and `helper-menu.sh` scripts are removed as their functionality is migrated to Ansible.
